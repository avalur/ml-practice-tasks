# ML Practice — web

Next.js app for the in-browser ML practice site. Student code runs **entirely in
the browser** via self-hosted Pyodide (`public/pyodide`); problem content comes
from the committed `public/content` bundle (generated by `python ../export_web.py`).

## Develop

```bash
pnpm install
pnpm dev            # http://localhost:3000
```

## Build & test

```bash
pnpm build
pnpm exec playwright install chromium   # first time only
pnpm exec playwright test               # e2e: in-browser runner + auth smoke
```

## Auth (local dev)

Auth.js v5 with GitHub + Google, **JWT sessions (no database yet)**. The app
builds and shows a “Sign in” button without credentials; actual sign-in works
only once they’re set.

```bash
cp .env.local.example .env.local
```

Then fill `.env.local`:

- `AUTH_SECRET` — `npx auth secret` (or `openssl rand -base64 33`)
- **GitHub** OAuth App (https://github.com/settings/developers) — callback
  `http://localhost:3000/api/auth/callback/github`
- **Google** OAuth client (Cloud Console → Credentials) — redirect
  `http://localhost:3000/api/auth/callback/google`
